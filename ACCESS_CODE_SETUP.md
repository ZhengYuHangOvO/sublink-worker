# 访问代码验证功能设置指南 ## 概述 本项目已添加访问代码验证功能，可以保护您的部署网站不被未经授权的访问。支持两种配置方式： - **环境变量方式**：通过 Cloudflare 后台设置环境变量 - **网页管理方式**：通过网页管理界面直接设置（推荐） ## 功能特点 - **可选功能**：不设置访问代码时，网站保持公开访问 - **多代码支持**：支持配置多个访问代码，用逗号分隔 - **Cookie 验证**：验证通过后使用 Cookie 存储验证状态（30 天有效期） - **智能路由保护**：只保护网页界面，不影响订阅链接等 API 功能 - **多语言支持**：支持中文、英文、波斯语、俄语 - **网页管理**：可通过网页界面直接设置访问代码，无需修改环境变量 ## 快速开始（推荐） ### 方式一：通过网页管理界面设置（无需配置环境变量） 1. 访问 `https://你的域名/admin` 2. 如果未设置管理员密码，可以直接设置访问代码 3. 输入新的访问代码，点击"保存访问代码" 4. 保存成功后，访问主页将需要输入访问代码 ### 方式二：设置管理员密码（增强安全性） 如果需要额外的安全保护，可以设置管理员密码： 1. 在 Cloudflare 后台添加环境变量 `MASTER_PASSWORD`（或 `VITE_MASTER_PASSWORD`） - 值：设置一个强密码，例如 `MySecureAdminPassword123` 2. 重新部署 Worker 3. 访问 `https://你的域名/admin` 4. 输入管理员密码进行验证 5. 验证通过后，可以设置访问代码 ## 环境变量设置 ### 在 Cloudflare 后台设置 1. 登录 [Cloudflare Dashboard](https://dash.cloudflare.com/) 2. 进入 Workers 页面 3. 选择你的 Worker 4. 点击"设置" > "环境变量" 5. 添加以下变量（任选其一或全部）： | 变量名 | 说明 | 示例值 | |---------------------|----------------------------------------------------------|---------------------------| | `MASTER_PASSWORD` | 管理员密码（用于保护管理页面） | `MySecureAdminPassword` | | `ACCESS_CODE` | 预设访问代码（优先级高于网页设置） | `user123` | | `VITE_MASTER_PASSWORD` | 管理员密码别名 | `MySecureAdminPassword` | | `VITE_ACCESS_CODE` | 访问代码别名 | `user123` | 6. 保存并部署 ### 多个访问代码 支持配置多个访问代码，用逗号分隔： ``` ACCESS_CODE=code1,code2,code3 ``` ## 受保护的路由 以下路由需要访问代码验证： - `/` - 主页 - `/config` - 配置保存 API - `/shorten-v2` - 短链接生成 - `/resolve` - 短链接解析 ## 不受保护的路由 以下路由**不需要**访问代码，确保订阅功能正常工作： - `/singbox` - SingBox 配置 - `/clash` - Clash 配置 - `/surge` - Surge 配置 - `/subconverter` - Subconverter 配置 - `/xray` - Xray 配置 - `/s/:code`, `/b/:code`, `/c/:code`, `/x/:code` - 短链接重定向 - `/admin` - 管理页面 - `/api/admin/*` - 管理 API - `/favicon.ico` - 网站图标 ## 使用说明 ### 首次使用 1. 访问 `https://你的域名/admin` 2. 直接输入新的访问代码并保存 3. 访问主页时会提示输入访问代码 4. 输入正确的访问代码即可进入 ### 设置管理员密码后 1. 访问 `https://你的域名/admin` 2. 输入管理员密码进行验证 3. 验证通过后，可以设置/修改访问代码 ### 清除访问代码 1. 访问 `https://你的域名/admin` 2. 在"新访问代码"输入框中留空 3. 点击"保存访问代码" 4. 访问代码保护将被移除 ## 清除验证状态 如果忘记了访问代码或需要清除验证状态： ### 方法 1：清除浏览器 Cookie - Chrome/Edge: 设置 > 隐私和安全 > Cookie 和其他网站数据 - Firefox: 设置 > 隐私与安全 > Cookie 和网站数据 ### 方法 2：使用开发者工具 ```javascript // 在浏览器控制台执行 document.cookie = "access_code=; expires=Thu, 01 Jan 1970 00:00:00 UTC; path=/;"; document.cookie = "admin_auth=; expires=Thu, 01 Jan 1970 00:00:00 UTC; path=/;"; ``` ### 方法 3：通过管理页面 如果设置了管理员密码，可以通过管理页面清除访问代码 ## 安全建议 1. **使用强密码**：访问代码和管理员密码应足够复杂，避免被猜测 2. **定期更换**：建议定期更换访问代码 3. **不要泄露**：不要将访问代码分享给未经授权的用户 4. **HTTPS 部署**：确保在 HTTPS 环境下部署，保护 Cookie 安全 5. **管理员密码保护**：设置管理员密码可以防止他人修改访问代码 ## 故障排除 ### 问题：设置访问代码后无法访问 - 检查环境变量是否正确设置 - 清除浏览器 Cookie 后重试 - 检查部署平台的环境变量配置 ### 问题：订阅链接无法使用 - 确认订阅链接路径不受保护（如 `/singbox`, `/clash` 等） - 检查是否有误配置其他路由 ### 问题：管理页面无法设置访问代码 - 检查是否配置了 KV 命名空间（Cloudflare Workers 需要 KV 存储） - 在 Cloudflare 后台为 Worker 绑定 KV 命名空间 ### KV 命名空间配置 如果管理页面提示"KV 存储不可用"，需要配置 KV 命名空间： 1. 在 Cloudflare Dashboard 创建 KV 命名空间 2. 在 Worker 设置中绑定 KV 命名空间 3. 在代码中确保 `runtime.kv` 可用 ## 配置优先级 访问代码的优先级顺序： 1. **环境变量 `ACCESS_CODE`** - 最高优先级，无法通过网页修改 2. **KV 存储的访问代码** - 可通过网页管理界面修改 如果同时设置了环境变量和 KV 存储，两者都会生效（用户可以访问任一代码） ## 开发说明 ### 添加新的受保护路由 在 `src/app/createApp.jsx` 中修改 `PROTECTED_PATHS` 数组： ```javascript const PROTECTED_PATHS = ['/', '/config', '/shorten-v2', '/resolve', '/your-new-path']; ``` ### 添加新的不受保护路由 在 `src/app/createApp.jsx` 中修改 `UNPROTECTED_PATHS` 数组： ```javascript const UNPROTECTED_PATHS = [ '/singbox', '/clash', // ... 其他路径 '/your-api-path', // 添加新的 API 路径 ]; ``` ## 技术实现 - **框架**：Hono.js - **验证方式**：中间件拦截 + Cookie 验证 - **Cookie 设置**：30 天有效期（访问代码），7 天有效期（管理员认证），SameSite=Strict - **存储方式**：KV 存储（Cloudflare Workers） - **安全性**：密码字段掩码，URL 编码存储，HTTPS Cookie ## 环境变量速查表 | 变量名 | 必需 | 说明 | |---------------------|------|----------------------------------------------------------| | `MASTER_PASSWORD` | 否 | 管理员密码，设置后可保护管理页面 | | `ACCESS_CODE` | 否 | 预设访问代码，优先级高于网页设置 | | `VITE_MASTER_PASSWORD` | 否 | 管理员密码别名 | | `VITE_ACCESS_CODE` | 否 | 访问代码别名 | ## 许可证 此功能遵循项目原有的 MIT 许可证。
